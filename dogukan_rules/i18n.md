# Internationalization (i18n) Rules

<critical>
## Critical Rule
**Always add translations to BOTH German (de/) AND English (en/) locale files.**
UI strings must support both languages.
</critical>

<checklist>
## Before Adding UI Strings
- [ ] Create key in both de/ and en/ locale files
- [ ] Use semantic key names (not literal translations)
- [ ] Group related keys by feature/page
- [ ] Consider pluralization needs
- [ ] Check for existing similar keys to reuse
</checklist>

<key-naming>
## Key Naming Conventions

### Hierarchical Structure
```json
{
  "common": {
    "submit": "Absenden",
    "cancel": "Abbrechen",
    "save": "Speichern",
    "delete": "Loeschen",
    "edit": "Bearbeiten",
    "loading": "Wird geladen..."
  },
  "forms": {
    "email": "E-Mail",
    "password": "Passwort",
    "name": "Name"
  },
  "validation": {
    "required": "Dieses Feld ist erforderlich",
    "email": "Bitte geben Sie eine gueltige E-Mail ein",
    "minLength": "Mindestens {min} Zeichen erforderlich"
  },
  "pharmacy": {
    "title": "Apotheke",
    "list": "Apothekenliste",
    "details": "Apothekendetails"
  }
}
```

### Key Naming Rules
```
feature.section.element

Examples:
pharmacy.form.nameLabel      -> "Apothekenname"
pharmacy.table.emailColumn   -> "E-Mail"
pharmacy.actions.delete      -> "Apotheke loeschen"
validation.errors.emailInvalid -> "Ungueltige E-Mail-Adresse"
```
</key-naming>

<usage>
## Usage in Vue Components

### Basic Usage
```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t, locale } = useI18n()
</script>

<template>
  <h1>{{ t('pharmacy.title') }}</h1>
  <Button :label="t('common.submit')" />
  <p>{{ t('common.loading') }}</p>
</template>
```

### With Interpolation
```json
// de/pharmacy.json
{
  "welcome": "Willkommen, {name}!",
  "itemCount": "Sie haben {count} Artikel"
}
```

```vue
<template>
  <p>{{ t('pharmacy.welcome', { name: userName }) }}</p>
  <p>{{ t('pharmacy.itemCount', { count: items.length }) }}</p>
</template>
```

### Pluralization
```json
// de/pharmacy.json
{
  "items": "Kein Artikel | Ein Artikel | {count} Artikel"
}

// en/pharmacy.json
{
  "items": "No items | One item | {count} items"
}
```

```vue
<template>
  <p>{{ t('pharmacy.items', itemCount) }}</p>
</template>
```

### Language Switching
```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { locale } = useI18n()

const changeLanguage = (lang: string) => {
  locale.value = lang
  localStorage.setItem('locale', lang)
}
</script>

<template>
  <select v-model="locale" @change="changeLanguage(locale)">
    <option value="de">Deutsch</option>
    <option value="en">English</option>
  </select>
</template>
```
</usage>

<setup>
## i18n Setup

```typescript
// locales/index.ts
import { createI18n } from 'vue-i18n'
import de from './de'
import en from './en'

export const i18n = createI18n({
  legacy: false,
  locale: localStorage.getItem('locale') || 'de',
  fallbackLocale: 'en',
  messages: { de, en }
})
```
</setup>

<anti-patterns>
## Anti-Patterns to Avoid
- Hardcoded strings in templates
- Missing translation in one language
- Using literal text as keys ("Submit" instead of "common.submit")
- Duplicating translations across files
- Not using interpolation for dynamic values
- Forgetting to update both locale files when adding new keys
</anti-patterns>
